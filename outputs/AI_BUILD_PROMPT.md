# Prompt For Other AI Tools

Build a professional production-ready college hostel management website named **NIT HostelHub** for an engineering/technical college in the Mangalore/Udupi side of Karnataka.

Requirements:
- Premium black and gold visual design.
- Smooth flow animations, responsive desktop/mobile layout, polished UI states.
- Bilingual support: English and Kannada.
- Public landing page plus separate student and admin portal sections.
- Working login flows, not static mockups.
- Student login should accept admission number or college Gmail.
- Demo admin credentials: `admin@hostelhub.edu.in` / `Admin@1234`.
- Demo student credentials: `NITHH2026CSE014` or `rahul.shetty26@nithostelhub.edu.in` / `Student@123`.
- Hostel types: boys hostel, girls hostel, co-ed/mixed hostel, PG/research scholar hostel.
- Student authentication concept: college ID login plus admin document verification.
- Student features: room browser, room picker from available rooms, priority score, fee tracking, leave/outpass management, complaint/grievance portal, mess/canteen information, notice board, visitor log.
- Admin features: student verification, document review, room management, grievance resolution, visitor log review, notice posting, leave approval.
- Nearby PG tracker: live-style student location, verified PG map and list side by side, search/filter, rent, rating, distance, safety tag, availability, contact action.
- PG tracker must cover Surathkal, Mangalore, Udupi, and Manipal, calculate distance from selected student location, and provide working route links.
- Student dashboard must show the student photo/avatar according to ID, basic profile details, hostel presence status, room, fee due, complaints, leave, visitors, and notices.
- Admin platform must track student verification, presence, room allocation, complaints, leave approvals, visitor log, notices, and hostel arrangements.
- Backend target: Supabase with auth, database, storage buckets for documents, RLS policies, and realtime updates.
- Deployment target: Vercel.

Use placeholder branding if no college logo is provided. Use realistic seeded data so the demo works immediately, but keep the architecture ready to connect to real Supabase tables and APIs.
